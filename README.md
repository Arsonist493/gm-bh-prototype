** 2D Bullet-Hell Logic Prototype **
This project is a 2D bullet-hell game developed using GameMaker, serving as my foundational entry into software development. It was designed to explore the boundaries of my technical capabilities by implementing complex game loops, entity interactions, and data-driven systems.

** Project Scope & Philosophy **
As my first software project, the focus was not on final product polish, but on figuring out the underlying logic of real-time systems. I am currently using this project as a sandbox to test how far I can push my problem-solving skills in terms of algorithm optimization and state-driven character behaviors.

**Core Technical Features **
-Dynamic Projectile System: A level-based weapon system where bullet patterns and properties evolve dynamically based on the player's progression while still giving the player multiple evolution paths to choose from.

- Skill & Shop Architecture: Implementation of a persistent data system that manages character upgrades, ability unlocks, and in-game economy.

- Entity AI: Logic for diverse enemy types with unique movement patterns and combat behaviors.

- State Machine Management: Handling complex transitions between player movement, spell-casting, and interaction states.

** Development Status (WIP) **
The project has successfully reached the final encounter phase. Current development is focused on:

- Boss Encounter Implementation: Designing a multi-phase, pattern-heavy boss fight to test advanced state management and collision optimization.

- Refining Ability Synergy: Balancing the interaction between different spells and shop-purchased skills.

** Technologies **
- Engine: GameMaker Studio 2

- Language: GML (GameMaker Language)

- Concepts: Object-Oriented Logic, State Machines, Collision Math.
